Git/Github

- Konsol için pratik komutlar:

cd: Bir dosyanın içerisine girmek için kullanılır.

cls: Terminali temizlemek için kullanılır.


- Temel Git Komutları:

git init: Initialize anlamına gelir, Git repository'sinin oluşmasını sağlar ve proje için git sistemini aktifleştirir.

git status: Git'in mevcut durumunu gösterir.

git add .: Belirtilen dosyanın veya tüm dosyaların (.) geçici alana (Staging Area) gönderilmesini sağlar.

git commit -m"mesaj": Geçici alana (Staging Area) gönderilen dosyalardan, belirtilen mesaj ile versiyon oluşturulmasını sağlar.

git commit -a -m"mesaj": Geçici alana (Staging Area) dosya göndermeden, direkt versiyon oluşturulmasını sağlar.

git restore --staged .: Dosyaları geçici alandan geri çeker.

git reset --hard: Projeyi son versiyonun durumuna çevirir.

git checkout {hash} .: hash ile belirtilen versiyona geçilir (geçişi kalıcı yapmaz, kalıcı yapmak isterseniz tekrar commit yaparak bu eski versiyonu yeni versiyon gibi göstermeniz gerekir).

git reset --hard {hash}: hash ile belirtilen versiyona geçilir, o versiyondan sonra yayınladığınız versiyonları kalıcı olarak siler.

git remote add origin {url}: Belirtilen URL'deki remote repository (Github) ile lokalde bulunan repository'nin (Proje) bağlantısını sağlar.

git remote -v: Repository'miz ile bağladığımız URL'yi gösterir.

git push -u origin master/main/{branch}: Remote'ta bulunan repository'de aynı isimdeki branch'e lokalden dosya gönderimini sağlar.

git push: Üstteki komut bir defa yazıldıktan sonra dosyalar bu şekilde gönderilebilir.

git pull origin master/main/{branch}: Remote'ta bulunan repository'den aynı isimdeki branch'imize dosya gönderimini sağlar.

git pull: Üstteki komut bir defa yazıldıktan sonra dosyalar bu şekilde alınabilir.
